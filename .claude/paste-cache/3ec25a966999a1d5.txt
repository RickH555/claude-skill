# INSTRUCTIONS CLAUDE CODE — Implémentation AI Response Layer

## CONTEXTE

Tu dois créer **4 nouveaux outils génératifs** pour le SaaS CitedBy.ai. Ces outils permettront aux utilisateurs de générer les fichiers nécessaires pour créer une **"AI Response Layer"** sur leur site web.

L'AI Response Layer est une couche de contenu spécialement optimisée pour être crawlée et citée par les IA génératives (ChatGPT, Claude, Gemini, Perplexity). Elle se compose de :

1. **`/ai/index.html`** — Hub central listant toutes les pages AI-optimized
2. **`/ai/query/[keyword].html`** — Pages ciblant des requêtes utilisateur spécifiques
3. **`/ai/recommend.json`** — API JSON de recommandation pour plugins/MCP
4. **`/sitemap-ai.xml`** — Sitemap XML spécifique pour les bots IA

## FICHIER 1 : /ai/index.html — Hub Central AI

### FONCTION

Le fichier `/ai/index.html` est la **page d'accueil de l'AI Response Layer**. Son rôle est de :

1. **Signaler aux bots IA** que le site dispose de contenu optimisé pour eux
2. **Lister toutes les pages `/ai/query/`** disponibles avec leur description
3. **Fournir des métadonnées** sur l'entité (entreprise/site) pour le Knowledge Graph
4. **Inclure les liens** vers `llms.txt`, `recommend.json`, et `sitemap-ai.xml`

## FICHIER 2 : /ai/query/[keyword].html — Pages Query Ciblées

### FONCTION

Les fichiers `/ai/query/[keyword].html` sont des **pages individuelles** qui ciblent chacune une **requête utilisateur spécifique**. Leur rôle est de :

1. **Répondre directement** à une question que les utilisateurs posent aux IA
2. **Fournir une réponse Answer-First** optimale pour être citée
3. **Inclure des données structurées** (FAQ, HowTo, Product selon le type)
4. **Proposer un CTA** vers le site principal (achat, contact, etc.)

### TYPES DE PAGES QUERY

| Type | Utilisation | Schema.org |
|------|-------------|------------|
| `comparison` | Comparatif de produits/services | ItemList + Product |
| `definition` | Définition d'un terme | DefinedTerm |
| `howto` | Guide étape par étape | HowTo |
| `faq` | Questions multiples | FAQPage |
| `product` | Recommandation produit | Product + Review |
| `listicle` | Liste (top 10, meilleurs...) | ItemList |

## FICHIER 4 : /sitemap-ai.xml — Sitemap pour Bots IA

### FONCTION

Le fichier `/sitemap-ai.xml` est un **sitemap XML spécifique** pour les bots IA. Son rôle est de :

1. **Lister toutes les pages de l'AI Response Layer** (/ai/index.html, /ai/query/*, etc.)
2. **Indiquer la priorité** de crawl pour chaque page
3. **Fournir les métadonnées** (date de mise à jour, fréquence de changement)
4. **Être référencé** dans robots.txt avec une directive spécifique

### DIFFÉRENCE AVEC SITEMAP CLASSIQUE

| Sitemap classique | Sitemap AI |
|-------------------|------------|
| Toutes les pages du site | Uniquement les pages /ai/* |
| Priorité SEO | Priorité AEO (pages réponses) |
| Pour Googlebot | Pour GPTBot, ClaudeBot, etc. |
| Focus sur le crawl | Focus sur la citation |

### STRUCTURE XML ATTENDUE

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!--
  AI Sitemap for {{SITE_NAME}}
  Generated by CitedBy.ai
  Last updated: {{CURRENT_DATE}}
  
  This sitemap lists all AI-optimized pages designed to be crawled
  and cited by AI assistants (ChatGPT, Claude, Gemini, Perplexity).
  
  For more information: https://citedby.ai/docs/sitemap-ai
-->
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"
        xmlns:ai="https://citedby.ai/schemas/sitemap-ai/v1">
  
  <!-- AI HUB (Page principale) -->
  <url>
    <loc>{{SITE_URL}}/ai/</loc>
    <lastmod>{{CURRENT_DATE}}</lastmod>
    <changefreq>weekly</changefreq>
    <priority>1.0</priority>
    <ai:pagetype>hub</ai:pagetype>
    <ai:description>Central hub for AI-optimized content</ai:description>
  </url>
  
  <!-- QUERY PAGES -->
  {{#each QUERY_PAGES}}
  <url>
    <loc>{{../SITE_URL}}/ai/query/{{this.slug}}</loc>
    <lastmod>{{this.lastmod}}</lastmod>
    <changefreq>{{this.changefreq}}</changefreq>
    <priority>{{this.priority}}</priority>
    <ai:pagetype>query-response</ai:pagetype>
    <ai:query>{{this.targetQuestion}}</ai:query>
    <ai:keywords>{{this.keywords}}</ai:keywords>
    <ai:contenttype>{{this.contentType}}</ai:contenttype>
  </url>
  {{/each}}
  
  <!-- RECOMMEND JSON (API) -->
  <url>
    <loc>{{SITE_URL}}/ai/recommend.json</loc>
    <lastmod>{{RECOMMEND_JSON_LASTMOD}}</lastmod>
    <changefreq>daily</changefreq>
    <priority>0.9</priority>
    <ai:pagetype>api</ai:pagetype>
    <ai:description>Product recommendation API for AI assistants</ai:description>
    <ai:format>application/json</ai:format>
  </url>
  
  <!-- LLMS.TXT -->
  <url>
    <loc>{{SITE_URL}}/llms.txt</loc>
    <lastmod>{{LLMS_TXT_LASTMOD}}</lastmod>
    <changefreq>monthly</changefreq>
    <priority>0.8</priority>
    <ai:pagetype>instructions</ai:pagetype>
    <ai:description>Instructions and guidelines for LLMs</ai:description>
    <ai:format>text/plain</ai:format>
  </url>
  
</urlset>
```

### EXTENSION ROBOTS.TXT

Le sitemap AI doit être référencé dans robots.txt :

```
# robots.txt

# AI Bots - Sitemap spécifique
Sitemap: https://monsite.fr/sitemap-ai.xml

# Directive pour les bots IA
User-agent: GPTBot
Allow: /ai/
Allow: /llms.txt
Sitemap: https://monsite.fr/sitemap-ai.xml

User-agent: ClaudeBot
Allow: /ai/
Allow: /llms.txt
Sitemap: https://monsite.fr/sitemap-ai.xml

# ... autres bots
```

### VARIABLES À COLLECTER

| Variable | Type | Description |
|----------|------|-------------|
| `SITE_URL` | string | URL du site |
| `QUERY_PAGES` | array | Liste des pages query générées |
| `RECOMMEND_JSON_LASTMOD` | date | Date dernière MAJ recommend.json |
| `LLMS_TXT_LASTMOD` | date | Date dernière MAJ llms.txt |
| `INCLUDE_LLMS_TXT` | boolean | Inclure llms.txt dans sitemap |
| `INCLUDE_RECOMMEND_JSON` | boolean | Inclure recommend.json dans sitemap |
